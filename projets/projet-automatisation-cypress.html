Le Contexte

Comprendre le challenge

Mission

Recrutement chez Eco Bliss Bath, e-commerce de produits cosmétiques éco-responsables. Julien, CTO, confie la mission d'automatiser les tests critiques pour gagner du temps sur les campagnes manuelles répétitives menées par Marie (QA manuelle).

Objectif

Trois axes d'automatisation : tests API pour valider produits et commandes, smoke tests pour éviter les régressions majeures, et un test end-to-end du parcours le plus risqué (ajout panier + paiement). Bonus : tester une faille XSS potentielle.

Livrables

  • Suite Cypress complète
  • Bilan de campagne
  • README documentation
  • Rapport anomalies

Détails clés

  • Client Eco Bliss Bath, e-commerce éco-responsable
  • CTO Julien
  • Rôle QA Automation Engineer
  • Durée 60h

Les Résultats

Mission accomplie

Suite Cypress complète pour Eco Bliss Bath : automatisation réussie des tests API, smoke tests, détection faille XSS et parcours end-to-end panier/paiement. Gain de temps significatif pour Marie (QA manuelle).

Tests API : GET /products valide structure JSON (id, nom, prix), POST /orders vérifie création commande avec status 201.
Smoke tests : page accueil charge en moins de 3s, liste produits affichée, formulaire contact présent.
Test XSS : injection script dans commentaire, vérification si le site échappe ou exécute le code (faille détectée).
E2E panier/paiement : parcours complet automatisé (sélection → panier → paiement → confirmation).
Sélecteurs robustes : data-testid pour éviter les cassures lors des changements UI, commandes custom.
Bilan de campagne : taux réussite 95%, 1 anomalie XSS critique détectée, recommandations échappement inputs.
