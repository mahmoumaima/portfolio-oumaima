Résumé du projet

Début d'alternance chez Booki, start-up de réservation d'hébergements et d'activités. La CTO Sarah lance la refonte du site avec une nouvelle interface conçue par Loïc, l'UI designer. Mission : intégrer la page d'accueil en HTML5 et CSS3 avec un design entièrement responsive.

Le projet nécessite une maîtrise complète du responsive design : maquette desktop fournie, adaptation tablette/mobile à réaliser avec Media Queries. Approche mobile-first non imposée mais encouragée pour faciliter l'adaptation multi-écrans.

  • Header & Navigation : logo Booki avec liens Hébergements/Activités, bordure bleue au survol (top desktop, bottom mobile).
  • Formulaire de recherche : 3 parties (icône, input, bouton) avec gestion texte/loupe selon format écran.
  • Filtres : 4 boutons interactifs avec changement de couleur au survol, disposition adaptative.
  • Cartes hébergements : grille Flexbox 3 colonnes (desktop) → 1 colonne (mobile), images object-fit, ombres portées.
  • Section populaires : cartes horizontales déjà fournies comme modèle, ordre d'affichage inversé mobile/desktop.
  • Activités à Marseille : grille 4 colonnes desktop avec hauteurs uniformes, images sémantiques via HTML.
  • Footer : 3 colonnes Flexbox, liens organisés avec espacement cohérent.

Méthodologie

Découpage de la maquette

Webinaire OpenClassrooms sur le découpage de maquette : identifier les blocs, choisir les bonnes balises sémantiques (header/nav/main/section/footer), définir les conteneurs Flexbox et anticiper le positionnement horizontal/vertical.

  • Identifier tous les éléments visibles
  • Regrouper par blocs logiques
  • Associer les balises HTML sémantiques
  • Planifier Flexbox (row vs column)

Résultats finaux

Page d'accueil Booki entièrement responsive : adaptation fluide desktop/tablette/mobile avec Flexbox et Media Queries. Interface fidèle à la maquette Figma, code sémantique validé W3C et performances optimisées.

  • Responsive design : 3 breakpoints gérés (desktop 1024-1440px, tablette ≤1024px, mobile <768px).
  • Flexbox mastery : navigation, filtres, cartes et footer organisés avec Flexbox, direction adaptative.
  • Cartes hébergements : grille 3 colonnes → 1 colonne mobile, images object-fit, ombres CSS subtiles.
  • Formulaire recherche : bouton texte desktop / icône loupe mobile, styles cohérents.
  • Accessibilité : tous les attributs alt renseignés, navigation clavier, contrastes respectés.
  • Code propre : HTML sémantique (header/nav/main/section/footer), CSS organisé, validation W3C 100%.
  • Performance : images optimisées, CSS épuré avec pixels pour margins/paddings et % pour widths.
