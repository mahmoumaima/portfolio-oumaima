Résumé du projet

Freelance pour Kasa, leader de la location d'appartements entre particuliers en France avec plus de 500 annonces. Laura, CTO, lance une refonte totale du site legacy (ASP.NET 15 ans d'âge) vers une stack moderne JavaScript avec NodeJS côté back-end et React côté front-end.

Paul, développeur back, s'occupe du backend. Mission confiée : développer l'ensemble de l'application front-end en React, en respectant les maquettes Figma et les coding guidelines. Le backend n'est pas encore prêt, donc travail avec fichier JSON de 20 annonces fournies.

  • React Router : configuration routes avec React Router pour navigation SPA entre pages (Accueil, À Propos, Logement, 404).
  • Page d'accueil : galerie de 20 cards logements, navigation vers page logement au clic sur card.
  • Page À Propos : affichage informations statiques avec composants Collapse réutilisables.
  • Page Logement : route dynamique avec ID logement, affichage toutes infos (titre, localisation, tags, host, rating, description, équipements), carrousel photos, collapses description/équipements.
  • Page 404 : redirection automatique vers 404 si ID logement inexistant ou route invalide.
  • Composants réutilisables : Gallery, Carousel, Card, Collapse, Rating, Tag à créer avec approche modulaire.
  • Responsive : maquettes desktop et mobile fournies, breakpoint tablette à définir logiquement.
  • Sass : utilisation recommandée pour styles, respect des coding guidelines (nommage, organisation fichiers).

Guidelines

Coding guidelines Kasa

Standards de code à respecter pour le projet :

  • React Router pour routing (navigation SPA)
  • Aucune librairie externe (pas de jQuery, Bootstrap...)
  • Create React App pour initialiser projet
  • React 18.3.1 minimum (version actuelle)
  • React Router v6 (dernière version stable)
  • Découpage logique en composants réutilisables
  • Un composant par fichier
  • Structure logique des fichiers
  • Utilisation Sass recommandée
  • Aucun warning dans console navigateur
Contraintes

Spécifications techniques détaillées

Comportements attendus par page :

  • Carrousel : si 1 seule image → masquer flèches, si plusieurs → clic flèche change image, numérotation type "1/4"
  • Collapse : fermé par défaut, clic ouvre/ferme avec animation rotation chevron 180°
  • Gallery : affichage 20 cards logements en grille responsive
  • 404 : route inexistante OU ID logement invalide → redirection page 404
  • Rating : affichage étoiles remplies (couleur) ou vides selon note sur 5

Résultats finaux

Application Kasa complète en React : navigation fluide avec React Router, 20 logements affichés en galerie, pages détaillées avec carrousel photos et composants interactifs, responsive sur tous formats.

  • React Router SPA : 4 routes configurées (/, /about, /logement/:id, /404), navigation sans rechargement.
  • Page Accueil : galerie 20 cards responsive, clic card → navigation vers page logement correspondante.
  • Page Logement : route dynamique avec ID, affichage toutes données (titre, localisation, tags, host, rating 5 étoiles, carrousel photos).
  • Carrousel photos : flèches navigation (masquées si 1 seule photo), numérotation "1/4", changement image au clic.
  • Composants Collapse : description et équipements dépliables, animation chevron 180°, fermés par défaut.
  • Page 404 : route inexistante ou ID invalide → redirection automatique vers page erreur 404.
  • Composants réutilisables : Card, Carousel, Collapse, Rating, Tag codés en modules indépendants.
  • Responsive complet : maquettes desktop/mobile respectées, breakpoint tablette défini logiquement.
  • Code quality : Sass organisé, un composant par fichier, aucun warning console, guidelines respectées.
