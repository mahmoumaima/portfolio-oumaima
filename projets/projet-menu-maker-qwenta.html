Résumé du projet

Agence Webgencia, mission pour Qwenta (client du portefeuille). Soufiane, Product Owner, confie la préparation technique du projet "Menu Maker" : application permettant aux restaurateurs de créer des menus en drag & drop, gérer allergènes et générer la carte PDF.

Rôle Product Owner pour ce projet : rédiger les spécifications techniques complètes, organiser le travail en user stories sur un board Kanban, et mettre en place un système de veille technologique pour rester à jour sur React, bases de données et sécurité.

  • Spécifications techniques : analyse specs fonctionnelles, choix solution technique argumentée (stack front/back, BDD relationnelle vs NoSQL), documentation architectures et justifications.
  • User Stories : découpage projet en user stories avec format "En tant que [rôle], je veux [action] afin de [bénéfice]", critères d'acceptation clairs pour chaque story.
  • Kanban : board Notion avec colonnes "To Do", "In Progress", "Done", priorisation user stories, assignment responsables développeurs.
  • Veille technologique : mise en place outil Feedly, sélection sources pertinentes (React, sécurité web, bases de données), suivi régulier tendances et best practices.
  • Présentation orale : soutenance devant Soufiane pour défendre choix techniques, expliquer organisation Kanban et démontrer pertinence système de veille.

Résultats finaux

Mission Product Owner complète pour Menu Maker Qwenta : spécifications techniques argumentées, organisation projet en user stories Kanban et système de veille technologique opérationnel.

  • Specs techniques complètes : choix stack justifié (React front, Node.js/Express back), comparatif MongoDB vs PostgreSQL argumenté, architecture API REST détaillée.
  • User Stories structurées : découpage fonctionnel complet format "En tant que restaurateur, je veux...", critères d'acceptation clairs pour chaque story.
  • Kanban Notion organisé : board avec colonnes To Do/In Progress/Done, priorisation MoSCoW (Must/Should/Could Have), assignment développeurs par expertise.
  • Veille technologique : Feedly configuré avec sources React (blog officiel, dev.to), sécurité web (OWASP), bases de données (MongoDB/PostgreSQL blogs).
  • Approche argumentée : chaque choix technique justifié (pourquoi React vs Vue, MongoDB vs PostgreSQL selon besoins métier).
  • Soutenance réussie : présentation orale devant Soufiane défendant les choix techniques, démonstration Kanban et pertinence système veille.
