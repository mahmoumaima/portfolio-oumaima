Résumé du projet

Recrutement chez Eco Bliss Bath, e-commerce de produits cosmétiques éco-responsables. Julien, CTO, confie la mission d'automatiser les tests critiques pour gagner du temps sur les campagnes manuelles répétitives menées par Marie (QA manuelle).

Trois axes d'automatisation : tests API pour valider produits et commandes, smoke tests pour éviter régressions majeures, et un test end-to-end du parcours le plus risqué (ajout panier + paiement). Bonus : tester une faille XSS potentielle sur les commentaires produits signalée en interne.

  • Tests API : GET /products (liste produits), POST /orders (création commande), validation réponses JSON et codes status HTTP.
  • Smoke tests : vérifications rapides post-déploiement (page accueil charge, produits s'affichent, formulaire contact existe).
  • Test XSS : injection script malveillant dans champ commentaire produit, vérification si le site échappe ou exécute le code.
  • Test E2E panier/paiement : parcours complet utilisateur (sélection produit, ajout panier, remplissage formulaire paiement, confirmation commande).
  • Bilan de campagne : rédaction rapport avec taux de réussite, anomalies détectées, captures d'écran et recommandations.

Résultats finaux

Suite Cypress complète pour Eco Bliss Bath : automatisation réussie des tests API, smoke tests, détection faille XSS et parcours end-to-end panier/paiement. Gain de temps significatif pour Marie (QA manuelle).

  • Tests API : GET /products valide structure JSON (id, nom, prix, description), POST /orders vérifie création commande avec status 201.
  • Smoke tests : page accueil charge en <3s, liste produits affichée, formulaire contact présent avec champs requis.
  • Test XSS : injection <script>alert('XSS')</script> dans commentaire, vérification si site échappe ou exécute (faille détectée).
  • E2E panier/paiement : parcours complet automatisé (sélection produit → ajout panier → formulaire paiement → confirmation), validation message succès.
  • Sélecteurs robustes : data-testid pour éviter cassures lors changements UI, commandes custom Cypress pour réutilisation.
  • Bilan de campagne : rapport avec taux réussite 95%, 1 anomalie XSS critique détectée, recommandations (échapper inputs utilisateur).
  • README complet : instructions installation, lancement tests, structure dossier cypress/, explications chaque fichier spec.
