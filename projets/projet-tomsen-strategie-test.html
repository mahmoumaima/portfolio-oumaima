Résumé du projet

Mission consultante test chez PayForge, entreprise développant solutions bancaires pour clients internationaux. Amandine, Product Owner de l'équipe Tomsen, confie l'élaboration de la stratégie de test pour nouvelle fonctionnalité permettant virements internationaux sécurisés.

Trois sprints de développement prévus sur 6 semaines (2 semaines par sprint). Mission : analyser exigences critiques, concevoir stratégie de test complète, créer cahier de recette avec cas de test détaillés, planifier tests par sprint et organiser campagne de non-régression finale.

  • Revue exigences : analyse document exigences Amandine, identification exigences testables, non testables et ambiguës, questions clarification.
  • Stratégie de test : objectifs qualité, périmètre in/out, identification risques critiques, définition critères entrée/sortie, approche tests (fonctionnel, sécurité, performance).
  • Analyse risques : matrice risques produit avec probabilité/impact, priorisation fonctionnalités à tester, focus sécurité transactions financières.
  • Cahier de recette : conception cas de test détaillés (préconditions, étapes, résultats attendus), couverture exigences critiques, organisation par user stories.
  • Planification tests : répartition cas de test sur 3 sprints, estimation charge avec abaque, jalons qualité, synchronisation avec équipe dev.
  • Non-régression : sélection tests critiques à rejouer fin cycle, automatisation prioritaire, rapport campagne finale avec métriques qualité.

Détails clés

  • Client PayForge, solutions bancaires internationales
  • Product Owner Amandine (équipe Tomsen)
  • Planning 3 sprints x 2 semaines (6 semaines total)
  • Rôle Consultante Test / QA Lead
  • Durée 40h
  • Livrables Stratégie test, cahier recette, matrice risques, planning tests

Résultats finaux

Stratégie de test complète pour Tomsen PayForge : revue exigences, analyse risques, cahier de recette détaillé et planification tests sur 3 sprints avec jalons qualité.

  • Revue exigences : analyse document Amandine, identification exigences testables (90%), non testables (5%), ambiguës (5%), questions clarification formulées.
  • Stratégie de test : objectifs qualité définis (sécurité transactions, fiabilité virements internationaux), périmètre in/out scope précisé.
  • Analyse risques : matrice probabilité/impact avec 15 risques identifiés, priorisation fonctionnalités critiques (validation montant, conversion devises, sécurité).
  • Cahier de recette : 45 cas de test détaillés couvrant exigences critiques, format standardisé (ID, user story, préconditions, étapes, résultats attendus).
  • Planning tests : répartition sur 3 sprints (Sprint 1: 15 cas, Sprint 2: 18 cas, Sprint 3: 12 cas), estimation charge avec abaque (120h total).
  • Jalons qualité : critères GO/NO GO par sprint (Sprint 1: 80% réussite, Sprint 2: 85%, Sprint 3: 95% + zéro bloquant).
  • Non-régression : sélection 20 tests critiques à rejouer fin cycle, recommandation automatisation prioritaire (Cypress/Selenium).
