Le Contexte

Comprendre le challenge

Mission

QA Tester dans une agence web développant le site de 724events, agence événementielle. Le chef de projet Jean-Baptiste confie la mission de lancer la campagne de tests sur la première version du site déjà livrée par l'équipe dev.

Objectif

Aller plus loin que les tests fonctionnels : analyser en profondeur chaque anomalie détectée avec les logs console navigateur, Swagger pour tester l'API, et créer des tickets Jira détaillés.

Livrables

  • Cahier de tests avec statuts (Notion)
  • Board Jira exporté CSV
  • Bilan de campagne de test

Détails clés

  • Client 724events, agence événementielle
  • Chef de projet Jean-Baptiste
  • Rôle QA Tester

Stack & Outils

Docker Jira Notion Swagger DevTools GitHub
Le Parcours

Phases de tests

01
Phase 1

Setup environnement

Installation Docker, lancement du site (localhost:3000) et de Swagger API (localhost:8080/api-docs), création compte Jira.

Docker Swagger Jira
02
Phase 2

Exécution cahier de recette

Lecture complète du cahier Notion, exécution méthodique des tests et notation des résultats avec identification des anomalies.

Notion Tests fonctionnels OK/KO
03
Phase 3

Analyse approfondie

Pour chaque bug : analyse logs console, test API avec Swagger, détermination de la criticité et création de tickets Jira détaillés.

DevTools API Testing Jira
04
Phase 4

Tests exploratoires & Bilan

2 sessions exploratoires (20 min + 30 min), minimum 2 anomalies, mise à jour cahier et rédaction du bilan de campagne GO/NO GO.

Exploratoires Bilan GO/NO GO
Les Résultats

Mission accomplie

Campagne de tests complète sur plateforme événementielle : cahier de recette exécuté, anomalies analysées en profondeur avec outils techniques (Swagger, logs console), tickets Jira détaillés et bilan de campagne rédigé.
