Résumé du projet

Intégration d'une agence web en tant que stagiaire QA Tester. L'agence travaille sur le site d'un cabinet d'avocats éco-responsables (JSE Avocats). Mission confiée par Sacha, référente QA : tester la première version du site WordPress en cours de développement.

Double approche de tests : exécuter les tests fonctionnels à partir d'un cahier de recette fourni (Notion), puis mener des sessions de tests exploratoires pour identifier les anomalies non prévues dans le cahier. L'environnement de test fonctionne en local via Docker.

  • Installation Docker : télécharger le conteneur, lancer l'environnement et vérifier l'accès au site local (localhost:8081).
  • Cahier de recette : lire l'intégralité du cahier Notion, exécuter les tests étape par étape, noter les résultats (OK/KO) et commentaires.
  • Tests exploratoires : 3 sessions thématiques (10-15 min chacune) - données (textes longs, caractères spéciaux), préférences (navigation privée, différents navigateurs, redimensionnement), tests libres sur l'interface.
  • Documentation anomalies : ajouter chaque anomalie découverte dans le cahier avec format "test exploratoire #X", étapes de reproduction et résultat obtenu.

Détails clés

  • Client JSE Avocats, cabinet d'avocats éco-responsables
  • Référente QA Sacha
  • Rôle Stagiaire QA Tester
  • Durée 30h
  • Sessions exploratoires 3 thèmes × 10-15 min (max 1h total)
  • Livrables Cahier de recette complété sur Notion avec anomalies documentées

Cahier de recette

Plan de tests Notion

Cahier de recette structuré sur Notion avec tous les cas de tests à exécuter. Format : cas de test, étapes à suivre, résultat attendu, colonnes pour noter le résultat (OK/KO) et les commentaires.

Méthodologie tests

Exécution du cahier de recette

Approche méthodique pour exécuter les tests fonctionnels :

  1. Lecture complète : prendre connaissance de tous les tests pour avoir une vision globale
  2. Exécution étape par étape : suivre le cahier et dérouler les étapes décrites
  3. Notation résultats : indiquer OK si conforme, KO si anomalie détectée
  4. Commentaires : pour chaque KO, préciser le résultat obtenu (ex: "Le lien XXX ne fonctionne pas")
Tests exploratoires

Sessions thématiques

Préparation et exécution de 3 sessions de tests exploratoires (10-15 min chacune) :

  • Session 1 - Données : tester avec textes très longs, caractères spéciaux, accentués dans les formulaires
  • Session 2 - Préférences : navigation privée, refus cookies, différents navigateurs, redimensionnement fenêtre, taille police
  • Session 3 - Tests libres : explorer l'interface sans contraintes, chercher des cas d'usage non prévus

Documentation : Pour chaque anomalie → ajouter "test exploratoire #1" dans cahier, décrire étapes de reproduction et résultat obtenu.

Résultats finaux

Campagne de tests complète sur le site JSE Avocats : cahier de recette entièrement exécuté avec statuts documentés, 3 sessions de tests exploratoires menées selon les thématiques définies, et toutes les anomalies détectées soigneusement documentées dans le cahier Notion.

  • Environnement maîtrisé : installation Docker réussie, site WordPress fonctionnel en local (localhost:8081).
  • Tests fonctionnels : cahier de recette Notion exécuté intégralement, chaque test noté (OK/KO) avec commentaires précis.
  • Tests exploratoires : 3 sessions thématiques réalisées (données, préférences, tests libres) dans le temps imparti (max 1h).
  • Anomalies documentées : chaque bug trouvé ajouté au cahier avec format "test exploratoire #X", étapes de reproduction détaillées.
  • Approche méthodique : lecture complète du cahier avant exécution, pas de tests redondants, focus sur la valeur ajoutée.
  • Curiosité QA : recherche de cas d'usage non prévus (ex: 2000 caractères dans champ email), différents navigateurs testés.
