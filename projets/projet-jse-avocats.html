Le Contexte

Comprendre le challenge

Mission

Intégration d'une agence web en tant que stagiaire QA Tester. L'agence travaille sur le site d'un cabinet d'avocats éco-responsables (JSE Avocats). Mission confiée par Sacha, référente QA.

Objectif

Double approche de tests : exécuter les tests fonctionnels à partir d'un cahier de recette fourni (Notion), puis mener des sessions de tests exploratoires pour identifier les anomalies non prévues.

Livrables

  • Cahier de recette complété
  • Tests exploratoires documentés
  • Anomalies détaillées
  • Étapes de reproduction

Détails clés

  • Client JSE Avocats, cabinet éco-responsable
  • Référente QA Sacha
  • Rôle Stagiaire QA Tester
  • Durée 30h

Stack & Outils

Docker WordPress Notion Tests fonctionnels Exploratoires GitHub
Le Parcours

Phases de tests

01
Phase 1

Installation Docker

Téléchargement du conteneur, lancement de l'environnement et vérification de l'accès au site local (localhost:8081).

Docker WordPress Local

Phase 2

Lecture complète du cahier Notion, exécution des tests étape par étape, notation des résultats (OK/KO) avec commentaires.

Phase 3

Tests exploratoires

3 sessions thématiques (10-15 min) : données (textes longs, caractères spéciaux), préférences (navigateurs, responsive), tests libres.

Exploratoires Sessions Thématiques
04
Phase 4

Documentation anomalies

Ajout de chaque anomalie dans le cahier avec format standardisé : étapes de reproduction et résultat obtenu.

Les Résultats

Mission accomplie

Campagne de tests complète sur le site JSE Avocats : cahier de recette entièrement exécuté avec statuts documentés, 3 sessions de tests exploratoires menées selon les thématiques définies, et toutes les anomalies détectées soigneusement documentées.

Environnement maîtrisé : installation Docker réussie, site WordPress fonctionnel en local (localhost:8081).
Tests fonctionnels : cahier de recette Notion exécuté intégralement, chaque test noté (OK/KO) avec commentaires précis.
Tests exploratoires : 3 sessions thématiques réalisées (données, préférences, tests libres) dans le temps imparti (max 1h).
Anomalies documentées : chaque bug trouvé ajouté au cahier avec format standardisé, étapes de reproduction détaillées.
Approche méthodique : lecture complète du cahier avant exécution, pas de tests redondants, focus sur la valeur ajoutée.
Curiosité QA : recherche de cas d'usage non prévus (ex: 2000 caractères dans champ email), différents navigateurs testés.
