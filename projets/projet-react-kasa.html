Le Contexte

Comprendre le challenge

Mission

Kasa, leader français de la location d'appartements entre particuliers avec plus de 500 annonces postées chaque jour, entreprend une refonte complète de sa plateforme web vieillissante (ASP.NET, plus de 10 ans) vers une stack complète en JavaScript avec NodeJS côté back-end et React côté front-end.

Objectif

Démarrer le projet React et développer l'ensemble de l'application : composants React, routes React Router, en suivant les maquettes Figma (responsives) et les coding guidelines de Kasa. Le backend est disponible sur GitHub et peut être lancé localement via Docker pour récupérer les données des propriétés.

Livrables

  • Application React complète avec Vite
  • Routing SPA avec React Router
  • Composants réutilisables (Gallery, Collapse, Slideshow)
  • Design responsive (desktop/mobile)
  • Tests unitaires (coverage 80% minimum)

Détails clés

  • Client Kasa, leader location appartements France (500+ annonces)
  • CTO Laura
  • Rôle Développeur Front-End Freelance

Les Résultats

Mission accomplie

Application Kasa complète en React avec Vite : navigation fluide avec React Router, récupération des propriétés via l'API backend (Docker), pages détaillées avec carrousel photos et composants interactifs, responsive conforme aux maquettes Figma, tests unitaires avec coverage 80%.

Routing SPA implémenté : Configuration des routes (Accueil, À Propos, Logement/:id, 404) avec navigation fluide sans rechargement de page.
Page d'accueil développée : Création des composants Banner et Card, intégration de l'API avec useState/useEffect, affichage dynamique des propriétés.
Page logement réalisée : Route dynamique avec paramètre ID, affichage complet des données (titre, localisation, tags, host, rating, équipements).
Carrousel photos créé : Navigation cyclique entre images, masquage automatique des flèches si image unique, hauteur constante maintenue.
Composant Collapse développé : Gestion des états ouvert/fermé, fermeture par défaut à l'initialisation, animations fluides au clic.
Gestion des erreurs mise en place : Redirection automatique vers page 404 pour routes inexistantes ou ID de propriété invalide.
Tests unitaires rédigés : Couverture de 80% minimum sur composants avec state (Slideshow/Collapse) et sans state (Banner/Card).
Design responsive intégré : Respect des maquettes Figma desktop et mobile, animations conformes au prototype.
Contraintes techniques respectées : Utilisation minimale de dépendances (Vite, React Router, SASS), respect des coding guidelines.
