Le Contexte

Comprendre le challenge

Mission

Agence Webgencia, mission pour Qwenta (client du portefeuille). Soufiane, Product Owner, confie la préparation technique du projet "Menu Maker" : application permettant aux restaurateurs de créer des menus en drag & drop, gérer les allergènes et générer la carte PDF.

Objectif

Rédiger les spécifications techniques complètes, organiser le travail en user stories sur un board Kanban, et mettre en place un système de veille technologique pour React, bases de données et sécurité.

Livrables

  • Spécifications techniques
  • Kanban Notion
  • Système veille Feedly
  • Présentation orale

Détails clés

  • Client Qwenta, solution digitale restauration
  • Agence Webgencia
  • Rôle Product Owner / Chef de projet technique
  • Durée 40h

Les Résultats

Mission accomplie

Mission Product Owner complète pour Menu Maker Qwenta : spécifications techniques argumentées, organisation projet en user stories Kanban et système de veille technologique opérationnel.

Spécifications techniques : choix stack justifié (React front, Node.js/Express back), comparatif MongoDB vs PostgreSQL argumenté.
User Stories structurées : découpage fonctionnel format "En tant que restaurateur, je veux...", critères d'acceptation clairs.
Kanban Notion organisé : colonnes To Do/In Progress/Done, priorisation MoSCoW, assignment développeurs par expertise.
Veille technologique : Feedly configuré avec sources React (blog officiel, dev.to), sécurité web (OWASP).
Approche argumentée : chaque choix technique justifié (pourquoi React vs Vue, MongoDB vs PostgreSQL selon besoins).
Soutenance réussie : présentation orale devant Soufiane défendant les choix techniques et démonstration Kanban.
