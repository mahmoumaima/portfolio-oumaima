Le Contexte

Comprendre le challenge

Mission

QA Engineer chez Eco Bliss Bath, start-up e-commerce de produits de beauté écoresponsables (produit phare : savon solide). Suite au bilan de campagne de tests manuels de Marie (QA), Fabio (CTO) demande d'automatiser les tests critiques avec Cypress.

Objectif

Automatiser avec Cypress en JavaScript : tests API (6 requêtes du swagger), smoke tests, vérification faille XSS dans l'espace commentaire, et 2 tests fonctionnels choisis parmi 3 scénarios critiques selon les besoins métier.

Livrables

  • Scripts Cypress automatisés
  • Bilan de campagne de test
  • README avec procédure de lancement
  • Dépôt GitHub public

Détails clés

  • Entreprise Eco Bliss Bath, start-up e-commerce beauté éco
  • CTO Fabio
  • Rôle QA Engineer

Stack & Outils

Cypress 13 Node.js JavaScript Docker API Testing GitHub
Le Parcours

Phases d'automatisation

01
Étape 1

Analyse des éléments du projet

Récupération du projet GitHub, lecture du bilan de campagne de Marie, exploration des documents et templates. Installation de Cypress et reproduction manuelle des tests.

GitHub Cypress Analyse
02
Étape 2

Rédaction des recommandations

Sélection des 2 tests fonctionnels parmi les 3 proposés selon critères : fréquence, criticité (impact financier/juridique/image), couverture et complexité.

Recommandations Priorisation Critères
03
Étape 3

Rédaction des scripts de test

Automatisation avec Cypress : tests des 6 requêtes API (swagger localhost:8081/api/doc), 2 tests fonctionnels critiques, smoke tests, et vérification faille XSS.

API Fonctionnels XSS
04
Étape 4

Analyse des résultats

Exécution des tests, analyse des échecs (anomalie ou flaky test?), identification du niveau (front-end/back-end), documentation des anomalies détectées.

Résultats Anomalies Analyse
05
Étape 5

Rédaction du bilan de campagne

Complétion du bilan avec anomalies détaillées (reproduction, correction), README en markdown avec procédures de lancement et génération rapport. Publication sur GitHub public.

Bilan README GitHub
Les Résultats

Mission accomplie

Suite Cypress complète pour Eco Bliss Bath : automatisation des tests API (6 requêtes), smoke tests, vérification faille XSS, et 2 tests fonctionnels critiques. Bilan de campagne avec anomalies détaillées et niveau identifié (front/back).

Tests API automatisés : 6 requêtes du swagger testées avec Cypress cy.request(), validation des codes HTTP et structures JSON.
Smoke tests implémentés : Vérifications rapides post-déploiement mises en place pour identifier les bugs critiques.
Faille XSS vérifiée : Test d'injection de script dans l'espace commentaire pour valider l'échappement du code.
Tests fonctionnels sélectionnés : 2 scénarios critiques choisis selon fréquence d'utilisation, criticité métier et couverture.
Anomalies identifiées et documentées : Bugs détectés avec niveau identifié (front-end/back-end) pour faciliter la correction.
Bilan de campagne rédigé : Document complet avec recommandations, contexte, objectifs, tests effectués et rapport d'anomalies.
